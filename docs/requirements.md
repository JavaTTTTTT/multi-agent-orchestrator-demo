# 需求分析

## 目标用户
科研人员（中文为主），日常阅读英文论文，需要快速理解论文核心内容。

## 核心功能
1. **论文解析** — 上传 PDF / 输入 URL，自动提取结构化内容
2. **多级摘要** — 一句话总结、核心发现、详细摘要
3. **方法论分析** — 技术方法、实验设计、优缺点评价
4. **多轮追问** — 基于论文内容的自由 QA 对话
5. **结构化报告** — 统一输出 Markdown 格式的解析报告

## 使用场景

| 场景 | 用户输入 | 期望输出 |
|------|----------|----------|
| 全量解析 | 上传 PDF / "帮我解析这篇论文" | 完整解析报告 |
| 快速摘要 | "总结一下这篇论文" | 多级摘要 |
| 方法聚焦 | "这篇论文用了什么方法" | 方法论分析 |
| 自由追问 | "attention 机制在文中怎么改进的" | 基于论文的回答 |

## 非功能需求
- 输出语言：中文（学术术语保留英文原文）
- 响应时间：全量解析 < 60s，追问 < 10s
- 支持的论文格式：PDF（优先），后续支持 URL
